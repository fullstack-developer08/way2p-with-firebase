Angular Lifecycle Hooks
Introduction
ngOnChanges It executes multiple times. First time it execute when component gets initialized and after that it execute when we bind any value to @input property from the parent component.
ngOnInit It executes only once when we initialized the component.
ngDoCheck It executes multiple times when any change detect by the component like button click, property change if it render the UI.
ngAfterContentInit It executes only once and when ng-content of the component get initialized.
ngAfterContentChecked It executes multiple times when any change detect in ng-content elements.
ngAfterViewInit It executes only once when component get initialized fully and rendered into UI.
ngAfterViewChecked It executes only once when component and child component get initialized fully and rendered into UI
ngOnDestroy When our component about to destroy. It execute just before destroy.
