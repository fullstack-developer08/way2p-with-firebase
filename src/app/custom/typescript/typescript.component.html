TypeScript is an open source pure object oriented programming language(means supports such as classes, interfaces, inheritance, generics, etc.). It is a strongly typed superset of JavaScript which compiles to plain JavaScript. It contains all elements of JavaScript. It is a language designed for large scale JavaScript application development, which can be executed on browser. The typescript is a language as well as a set of tools. Typescript is the ES(ECMAScript)6 version of JavaScript with some additional features.

Typescript cannot run directly on the browser. It needs a compiler to compile the file and generate it in JavaScript file, which can run directly on the browser. The Typescript source file is in ".ts" extension. Typescript uses TSC(Typescript Compiler), which convert Typescript code (.ts file) to JavaScript (.js file).

Advantages

  • Typescript support static typing, Strongly type, Modules, Optional Parameters etc.
  • TypeScript support object oriented programming features such as classes, interfaces, inheritance, generics, etc.
  • Type is simple and easy to learn.
  • TypeScript is fast.
  • TypeScript provides the error -checking feature at compilation time. It will compile the code and if any error found, then it highlighted the mistakes before the script is run.
  • TypeScript support all the JavaScript libraries because it is the superset of JavaScript. TypeScript Support reusability because of inheritance.
When application code extends in JavaScript, then it becomes difficult to maintain and complex to understand. And JavaScript was not able to fulfil the requirement of Object Oriented programming language which prevent JavaScript from succeeding at the enterprise level. To overcome these problem TypeScript comes into picture.

TypeScript Code is converted into Plain JavaScript Code : TypeScript code is not understandable by the browsers that is why if the code is written in TypeScript then it is compiled and converted the code i.e. translate the typescript code into JavaScript. This process is known as Transpiled with the help of JavaScript Code, browsers are able to read the code and display.

JavaScript is a subset of TypeScript: Whatever code is written in JavaScript can be converted into TypeScript by simply changing the extension from .js to .ts.

TypeScript can be used anywhere: TypeScript code can run any browser, devices or in any operating system. TypeScript is not specific to any Virtual-machine etc.

TypeScript Support JS libraries: With the support of TypeScript, developers can use existing JavaScript code, incluse popular JavScript libraries, and can be called other JavaScript code too.

TypeScript supports OOPs concept.

Advantage of TypeScript over JavaScript:

  • TypeScript always highlights errors at compilation time during the time of development, whereas JavaScript points out errors at the runtime.
  • TypeScript supports strongly typed or static typing, whereas this is not in JavaScript.
  • TypeScript runs on any browser or JavaScript engine.
  • Great tooling supports with intellisense which provides active hints as the code is added.
  • It has a namespace concept by defining by module.

Disadvantage of TypeScript over JavaScript:

  • 1. TypeScript takes a long time to compile the code.
  • 2. TypeScript does not support abstract classes.
  • If we run the TypeScript application in the browser, a compilation step is required to transform TypeScript into JavaScript.

The TypeScript language is internally divided into three main layers. Each of these layers is divided into sublayer or components.

These layers are:

Language
The TypeScript Compiler
The TypeScript Language Services

Language:

It features the TypeScript language elements. It comprises elements like syntax, keywords, and type annotations.

The TypeScript Compiler:

The TypeScript Compiler (TSC) transform the TypeScript program equivalent to its JavaScript code. It also performs the parsing, and type checking of our TypeScript code to JavaScript code.

App.ts ------->tsc app.ts------>app.js

Browser doesn't support the execution of TypeScript code directly. So the program written in TypeScript must be re-written in JavaScript equivalent code which supports the execution of code in browser directly. To perform this, TypeScript comes with TypeScript compiler named "tsc". The current version of TypeScript compiler supports ES6 by default. It compilers the source code in any module like ES6, SystemJS, AMD etc.

We can install the TyScript compiler by locally. Globally, or both with any npm package. Once installation completes, we can compile the TypeScript file by running the tsc command on the command line.

Example: tsc helloworld.tsc // it compiles the TS file hellowworld.ts into the helloworld.js file.

The TypeScript Language Services

The language service provides information which helps editors and other tools to give better assistance Features such as automated refactoring and intellisense. It exposes an additional layer around the core-compiler pipeline. It supports some standard typical editor operators like code formatting and outlining, colorization, statement completion, signature help, etc.

TypeScript Variables

TypeScript follows the same rules as JavaScript for variable declarations.
Variable can be declared using: var, let, and const

Var Declaration:
Variable in TypeScript can be declared using var keyword, same as in JavaScript. The scoping rules remains the same as in TypeScript.

Let Declaration:
To solve problems with var declarations, ES6 introduced two new types let and const variable declarations in JavaScript .

Variable Declaration using let:
The let declaration follow the same syntax as var declarations. Unlike variables declared with var, variables declared with let have a blocked-scope. This means that the scope of the let variables is limited to their containing block, e.g. function if else block or loop block.

Const Declaration:
Variables can be declared using const similar to var or let declarations. The const makes a variable a constant where its value can not be changed. Const variables have the same scoping rules as let variables.
