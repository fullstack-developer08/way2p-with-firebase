In Javascript, Object have a special hidden property [[prototype]](as named in the specification), that is either null or references another object. The object is called a prototype.

In the above attached image we can see highlighted __proto__ hidden property. This property we can use to set and get the object.

Using __proto__ property creating admin object from user object.

__proto__ is a getter/setter for [[prototype]]. __proto__ it is not same as [[prototype]]. It is a getter, setter for it.

If we will see attached image, there we can see user property are set into __proto__ property. since it is hidden property so we can directly access this property.

Here we can say user object is the prototype of admin object.

Or we can say admin is prototypically inherits from user.

So If user has lot of properties and methods then they will become automatically available in admin, such property are called "inherited".

If we have a method in user, it can be called on admin.

Hoisting is a default behaviour of javascript where all the variables and function declarations are moved on top.
This means that irrespective of where the variables and functions are declared, they are moved on top of the scope.
Note1: Variable initializations are not hoisted, only variable declarations are hoisted.
Note2: To avoid hositing, we can run javascript in strict mode by using "use strict" on top of the code.

Dynamic Typing: In javascript we will not declare type of the variable. Based on assigned value it will assign the type dynamically. so that Javascript is dynamically typed language.

Example

    var map = new Map();
