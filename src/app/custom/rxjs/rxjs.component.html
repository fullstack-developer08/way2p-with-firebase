Rxjs
  • Rxjs stands for Reactive Extension for Javscript.
  • It is a library for reactive programming using observables.
  • Reactive programming is an asynchronous programming that, concerned with data stream and the propagation of change.
  • Rxjs provides an implementation of the Observable and This Rxjs library also provides utility function for creating and working with observables.
  • It makes easier to compose asynchronous or callback-based code.
Pipe() function
  • Pipe let you combine multiple functions into a single function.
  • We can use pipe function to link operators together.
  • The pipe() function is also a method on the Rxjs Observable so you can use this sorter form to define the same operation.

from is used to convert promise and observable into the observable.

  • from operator can be used to convert a promise to an observable!
  • For arrays and iterables, all contained values will be emitted as a sequence!
  • This operator can also be used to emit a string as a sequence of characters!
  • of operator can take multiple arguments, but from operator take one argument promise, array or string.
Example - 1 
    const employees = [
        {{'{'}}name: 'xyz1'},
        {{'{'}}name: 'xyz2'}
    ];


    const promiseOfEmployees: Promise<Object> = Promise.resolve(employees);
    const promiseOfEmployees$: Observable<Object> = from(promiseOfEmployees);

    promiseOfEmployees$.subscribe(e => console.log(e));

    // convertering oberservable into observertion(it has no uses in real time)

    const person2$: Observable<Object> = from(person$);

    person2$.subscribe(e => console.log(e));
Example - 2 (Converting string into sequence of char) 
from('string').subscribe(s=>console.log(s))
Example - 3 (Converting array into sequence of values) 
from([1,2,3,4,5,6,7,8]).subscribe(s=>console.log(s))

interval operator

Creates an Observable that emits sequential numbers every specified interval of time.

interval operator needs only one argument, that is interval of time.

Example 
import {{'{'}} interval } from 'rxjs';
import {{'{'}} take } from 'rxjs/operators';

const numbers = interval(1000);

const takeFourNumbers = numbers.pipe(take(4));

takeFourNumbers.subscribe(x => console.log('Next: ', x));

// Logs:
// Next: 0
// Next: 1
// Next: 2
// Next: 3

timer operator

Creates an Observable that starts emitting after an dueTime and emits ever increasing numbers after each period of time thereafter.

It is same like interval but it takes two arguments, first is dueTime and second is interval of time.

If we will provide only one argument that is dueTime then it will take only 1 element from the observable.

Example - 1

Emits ascending numbers, one every second (1000ms), starting after 3 seconds 

import {{'{'}} timer } from 'rxjs';

const numbers = timer(3000, 1000);
numbers.subscribe(x => console.log(x));
Example - 2

Emits one number after five seconds 

import {{'{'}} timer } from 'rxjs';

const numbers = timer(5000);
numbers.subscribe(x => console.log(x));

{{tabs[5]}}

concat operator

Creates an output Observable which sequentially emits all values from given Observable and then moves on to the next.

Example 
import {{'{'}} concat, interval } from 'rxjs';
import {{'{'}} take } from 'rxjs/operators';

const timer1 = interval(1000).pipe(take(10));
const timer2 = interval(2000).pipe(take(6));
const timer3 = interval(500).pipe(take(10));

const result = concat(timer1, timer2, timer3);
result.subscribe(x => console.log(x));

// results in the following:
// (Prints to console sequentially)
// -1000ms-> 0 -1000ms-> 1 -1000ms-> ... 9
// -2000ms-> 0 -2000ms-> 1 -2000ms-> ... 5
// -500ms-> 0 -500ms-> 1 -500ms-> ... 9

{{tabs[6]}}

merge operator

Creates an output Observable which concurrently emits all values from every given input Observable.

Example 
import {{'{'}} merge, interval } from 'rxjs';
import {{'{'}} take } from 'rxjs/operators';

const timer1 = interval(1000).pipe(take(10));
const timer2 = interval(2000).pipe(take(6));
const timer3 = interval(500).pipe(take(10));

const result = merge(timer1, timer2, timer3);
result.subscribe(x => console.log(x));

                                                    debounceTime
                                                
                                            

                                            

                                                

                                                    Mostly debounceTime is used with input keyup event. When we press the key and leave the keyboard button that time we are calling keyup event. keyup event will be get called for each and every keyboard key, if we will call API for each and every keyup
                                                    that will reduce the application performance. To avoid this we can use debounceTime. debounceTime will be used with some interval time, this interval time avoid every keyup event within this interval
                                                    of time and it will dispatch the last keyup with input data if interval of time will be passed.

                                                    distinctUntilChanged() only emit when the current value is different than the last.

@Component({{'{'}}

  styleUrls: ["./app.component.css"]

    const keyupObs1 = fromEvent(this.textInput.nativeElement, "keyup").pipe(
      map((event: any) => event.target.value)

    keyupObs1.pipe(debounceTime(700)).subscribe(data => {{'{'}}
      this.textsOfDebounceTime.push(data);

    const keyupObs2 = fromEvent(this.textInput.nativeElement, "keyup").pipe(
      map((event: any) => event.target.value)

        debounceTime(700),
        distinctUntilChanged()

        this.texts.push(data);
      });
  }
}

                                                Rxjs Subject
                                            

                                            

                                                
    
                                                    
  • A subject is like an Observable.
    • 
                                                    
  • Subject are like EventEmitters: they maintain a registry of many listeners.
                                                    
    • 
                                                    
  • Every subject is an observable and we can subscribe to it.
    • 
                                                    
  • Every subject is an observer so we can use next() to pass the data.
                                                    
    • 
                                                

                                            

                                        


                                        

                                            

                                                BehaviourSubject & Subject
                                            

                                            

                                                
                                                    BehaviourSubject will return the initial value or the current value
                                                    on Subscription
                                                
                                                
var subject = new Rx.BehaviorSubject(0);  // 0 is the initial value

subject.subscribe({{'{'}}
    next: (v) => console.log('observerA: ' + v)  // output initial value, then new values on `next` triggers
});

subject.next(1);  // output new value 1 for 'observer A'
subject.next(2);  // output new value 2 for 'observer A', current value 2 for 'Observer B' on subscription

subject.subscribe({{'{'}}
    next: (v) => console.log('observerB: ' + v)  // output current value 2, then new values on `next` triggers
});

subject.next(3);
                                              

                                                
With output:

                                                
observerA: 0
observerA: 1
observerA: 2
observerB: 2
observerA: 3
observerB: 3  
                                              

                                                


                                                
                                                    Subject doesnot return the current value on Subscription. It
                                                    triggers only on .next(value) call and return/output the value

subject.next(1); //Subjects will not output this value

subject.subscribe({{'{'}}
    next: (v) => console.log('observerA: ' + v)
});

    next: (v) => console.log('observerB: ' + v)
