This feature was added in ES6. Spread and Rest operator are really the same thing. three periods or dots is both the spread and rest operators because they are really two side of the same thing. It's ability to take an array and split it into its individual elements or take a bunch of elements and combine them into a single array.

Spread Operator
Copy and extend Array using spread operator 
const names = ['Arif', 'Yash'];

console.log(names);

// push return length of the array so we will avoid this instead we will use concat
console.log(names.push('Sidarth')); 

// concat return the new array after concatnating, it will not update the existing variable
console.log(names.concat('Rohit')); 

// if you we see here our Rohit value missed in the array because of concat
console.log(names); 

// here we have spread operator and it is equal to the concat of above line
console.log([...names, 'Rohit']); 

console.log(['Facebook', ...names]);

Let us first talk about the normal function, whenever we just make a normal function what we do is we make a function and give it a name and then whenever we call it either just runs a bunch of code or if it has a return value then it runs up to the return statement and then gives you back a value.

Let us talk about generators, generators function they look a lot like we know just a normal function but the thing is they actually don't execute all the code at once like a normal function, what they do is say that you had 10 different lines in your generator, you could say run the first two lines and then pause and wait until i tell you to go on now once i tell you run the next three lines and then pause so it pretty much allows you to make function with like checkpoints in them.

checkpoints will be determined by yield keyword so basically means pause. I mean that's what it means in real life too.

next function is used to execute the execution of generator till the yeild field. If we will next time then it will start the execution after the first yeild and stop the execute on second yeild.

1. What is the difference between let and var keywords?

var:

-> var was there is JS from the beginning.
-> var has function scope.
-> var gets hoisted. (it means if try to get value of variable before declaration it will undefined)

let:

-> let was introduced in ES2015/ES6.
-> let has block scope.(which means variable defined with let keyword will die end of the block or garbage collected)
-> let does not get hoisted.(it means if we try to get value from before decalre variable it will return error in console like xyz is not defined)
3. What is the difference between let and const keywords?
Both is used to define variables
let
-> let we can assign the value as many no of times
-> It has block scope.
const
-> After first assignment of value we can not reassign the value again
-> It has function level scope.
-> If we declare variable with const but not initialized then it will give error like missing initializer
-> If we are creating variable as const but it type of Array or object then we can change the value but we can reassign the value.
